## Tema
Estimando User Stories

## Data da Anotação

## Objetivo da Aula
Compreender como estimar o esforço das User Stories, permitindo melhor planejamento das entregas e organização do trabalho do time

### O que são estimativas

Estimativas representam uma previsão do esforço necessário para desenvolver uma tarefa

No contexto ágil, o foco não é tempo exato, mas sim:
- complexidade  
- esforço  
- incerteza  

O material reforça que o desenvolvimento ágil trabalha com:
- adaptação  
- entregas contínuas  
- evolução constante  

### Estimativa de User Stories

As User Stories são estimadas para:
- planejar sprints  
- organizar o backlog  
- prever entregas  

A estimativa ajuda o time a entender o tamanho do trabalho antes de executá-lo

### Sequência de Fibonacci

Uma das técnicas mais utilizadas para estimativa é a sequência de Fibonacci

#### Padrão:
1, 2, 3, 5, 8, 13, 20

Essa sequência é usada porque:
- os intervalos aumentam progressivamente  
- refletem a incerteza conforme o esforço cresce  
- evitam falsa precisão  

### Como utilizar Fibonacci

Cada número representa o tamanho da tarefa

Exemplo:
- **1**        -> tarefa muito simples  
- **3**        -> tarefa média  
- **8**        -> tarefa complexa  
- **13 ou 20** -> tarefa muito grande  

Histórias muito grandes devem ser quebradas em partes menores

### Tamanho máximo das histórias

O material e a prática ágil indicam que:

- histórias não devem ser muito grandes  
- itens muito grandes (ex: 13 ou 20) devem ser divididos  
- isso facilita entrega contínua e melhor controle  

### Estimativa em equipe

A estimativa deve ser feita de forma colaborativa

O material reforça:
- importância da comunicação  
- envolvimento do time  
- troca de conhecimento  

O time discute as histórias e chega a um consenso sobre o esforço

### Benefícios das estimativas

- melhor planejamento  
- organização do backlog  
- previsibilidade  
- identificação de complexidade  
- redução de riscos  

### Relação com Agilidade

Estimativas estão ligadas aos princípios ágeis:

- **Indivíduos e interações**   -> decisão em equipe  
- **Responder a mudanças**      -> estimativas podem mudar  
- **Software em funcionamento** -> foco em entregas frequentes  

## Resumo

Estimativas são essenciais para planejar o desenvolvimento de User Stories. Utilizando técnicas como a sequência de Fibonacci, o time consegue avaliar o esforço das tarefas de forma colaborativa, evitando itens muito grandes e garantindo entregas mais eficientes e alinhadas aos princípios ágeis
