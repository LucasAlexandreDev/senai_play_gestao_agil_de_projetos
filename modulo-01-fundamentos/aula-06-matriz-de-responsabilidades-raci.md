## Objetivo da Aula
Compreender como montar uma matriz de responsabilidades (RACI), organizando as atividades e definindo claramente o papel de cada membro do time

### Revisão — Aula 03

A matriz RACI já foi apresentada na Aula 03 (Papéis e Responsabilidades), sendo uma ferramenta que define quem faz o quê dentro do projeto

Sigla:
- **R — Responsible** -> quem executa  
- **A — Accountable** -> quem aprova / decide  
- **C — Consulted**   -> quem é consultado  
- **I — Informed**    -> quem é informado  

 Nesta aula, o foco é na construção da matriz, não apenas no conceito

### Como montar uma Matriz RACI

A matriz é construída em formato de tabela, onde:

- **Linhas**  -> representam as atividades ou tarefas  
- **Colunas** -> representam os membros do time  

Cada cruzamento (célula) recebe uma letra (R, A, C ou I), indicando o papel da pessoa naquela atividade

#### Passo a passo:

1. Definir as atividades do projeto  
2. Listar todos os integrantes do time  
3. Montar a tabela (atividades x pessoas)  
4. Atribuir os papéis (R, A, C, I) para cada tarefa  
5. Revisar com o time para garantir alinhamento  


### Ferramentas utilizadas

A matriz pode ser construída com diferentes ferramentas, como:

- Excel  
- Google Sheets  
- Miro (destacado na aula)  

Essas ferramentas ajudam na visualização e colaboração do time

### Boas práticas

- Cada atividade deve ter pelo menos **1 Responsible (R)**  
- Deve haver apenas **1 Accountable (A)** por tarefa  
- Evitar excesso de pessoas como **Consulted (C)**  
- Garantir que todos os envolvidos estejam alinhados  

### Importância da Matriz RACI

A utilização da matriz permite:
- clareza nas responsabilidades  
- redução de conflitos  
- melhor comunicação  
- organização do trabalho  

Também evita dúvidas como:
- “quem faz isso?”  
- “quem aprova?”  
- “quem precisa saber?”
