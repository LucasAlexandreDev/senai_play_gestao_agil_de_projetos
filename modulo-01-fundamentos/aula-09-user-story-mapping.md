## Objetivo da Aula
Compreender como organizar e estruturar um backlog utilizando User Story Mapping, garantindo visão clara do produto e melhor planejamento das entregas

### Backlog do Produto

O backlog é uma lista organizada de tudo que precisa ser desenvolvido no produto

Ele contém:
- funcionalidades  
- melhorias  
- correções  

No contexto ágil (conforme o material), o backlog deve ser:
- dinâmico  
- priorizado  
- alinhado com o valor para o cliente  

### O que é User Story Mapping

User Story Mapping é uma técnica visual para organizar o backlog com foco na experiência do usuário

Ela permite:
- visualizar o produto como um todo  
- organizar as funcionalidades por fluxo de uso  
- priorizar entregas de forma estratégica  

Essa abordagem está alinhada com os princípios ágeis do material, como:
- foco no cliente  
- colaboração  
- entrega contínua de valor 

### Como montar um User Story Mapping

A estrutura é organizada em níveis:

#### 1. Atividades (nível alto)
Representam as grandes ações do usuário no sistema

Exemplo:
- acessar sistema  
- realizar cadastro  
- efetuar compra  

#### 2. Tarefas (nível intermediário)
Detalham as etapas dentro de cada atividade

#### 3. User Stories (nível detalhado)
São pequenas entregas que geram valor ao usuário

### Organização do Backlog

No User Story Mapping, o backlog é organizado:

- **Horizontalmente** -> fluxo do usuário (jornada)  
- **Verticalmente**   -> prioridade (mais importante primeiro)  

Isso permite definir:
- o que será feito primeiro  
- o que pode ser entregue depois  

### Releases e Entregas

A técnica também permite dividir o trabalho em entregas menores (releases)

Exemplo:
- primeira versão (MVP)  
- melhorias futuras  
- funcionalidades adicionais  

Isso está alinhado com o material, que destaca:
- entregas contínuas  
- adaptação às mudanças  
- foco em valor  

### Ferramenta utilizada: Miro

O Miro é utilizado para criar o User Story Mapping de forma visual e colaborativa

Com ele é possível:
- criar cards (post-its)  
- organizar atividades e histórias  
- mover itens facilmente  
- colaborar em equipe em tempo real  

### Benefícios

- visão clara do produto  
- melhor organização do backlog  
- priorização eficiente  
- foco na experiência do usuário  
- facilita planejamento das sprints  

## Resumo

O User Story Mapping é uma técnica visual que organiza o backlog com base na jornada do usuário. Utilizando ferramentas como o Miro, o time consegue estruturar atividades, tarefas e histórias de forma clara, priorizando entregas e garantindo valor contínuo ao cliente, conforme os princípios ágeis apresentados no material
